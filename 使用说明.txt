================================================================================
    Minecraft AFK 挂机互保脚本 - 使用说明
================================================================================

【简介】
  在一台电脑上同时运行两个 Minecraft 客户端时，本脚本通过本地 TCP 互联
  实现双实例互保：一方掉线（进程退出/崩溃/网络断开），另一方自动结束
  自己的 MC 客户端，避免单角色在线被怪物杀死或物资丢失。

【核心特性】
  ● 双向 TCP 心跳 + 告别消息协议（PEER_DOWN / SHUTDOWN / ALIVE）
  ● 服务器连接断开检测：MC 掉线但进程未退出时也能触发保护
  ● 多级进程终止：terminate → kill → taskkill 兜底
  ● 掉线后自动重启 Minecraft（需配置 --restart-command）
  ● 启动宽限期 90s：允许两个实例先后启动，期间不触发超时判定
  ● 指数退避重连：2s → 4s → 8s → ... → 30s
  ● TCP Keep-Alive + TCP NODELAY 优化
  ● Webhook 通知支持（Discord / 企业微信等）
  ● 配置文件 config.json 集中管理所有参数
  ● 日志输出到控制台 + 文件（可选）
  ● 宽泛的 MC 进程识别（支持原版/Forge/Fabric/各类启动器）
  ● psutil 自动安装 + netstat 回退检测

【快速开始】
  1. 安装依赖：pip install -r requirements.txt
     （或直接运行，脚本会自动安装 psutil）
  2. 在同一台电脑上启动两个 Minecraft 客户端
  3. 双击运行 start_instance_a.bat（对应第一个 MC 窗口）
  4. 双击运行 start_instance_b.bat（对应第二个 MC 窗口）
  5. 两个 CMD 窗口保持开启即可
  6. Ctrl+C 可安全退出（会发送 SHUTDOWN 通知对方）

【运行方式】

  方式一：使用批处理文件（推荐）
    start_instance_a.bat     → 实例 A（端口 18888，对端 18889）
    start_instance_b.bat     → 实例 B（端口 18889，对端 18888）

  方式二：使用 --instance 快捷参数（自动读取 config.json）
    python afk_monitor.py --instance a
    python afk_monitor.py --instance b

  方式三：全自动模式（自动检测 MC 进程）
    python afk_monitor.py --port 18888 --peer-port 18889 --auto --auto-index 0
    python afk_monitor.py --port 18889 --peer-port 18888 --auto --auto-index 1

  方式四：手动指定 PID
    python afk_monitor.py --port 18888 --peer-port 18889 --pid <MC进程PID>

  查看所有 Minecraft 进程：
    python afk_monitor.py --list

【命令行参数说明】
  --port PORT             本实例监听端口号（默认 18888）
  --peer-port PORT         对方实例监听端口号（默认 18889）
  --pid PID                手动指定要监控的 MC 进程 PID
  --auto                   全自动检测 MC 进程（需配合 --auto-index）
  --auto-index N           自动模式下选择第几个进程（0=第1个，默认 0）
  --instance {a,b}         使用预设实例快捷配置
  --config PATH            配置文件路径（默认 config.json）
  --list                   列出所有 Minecraft 进程后退出
  --heartbeat-interval N   心跳间隔秒数（默认 3，覆盖配置文件）
  --heartbeat-timeout N    心跳超时秒数（默认 15，覆盖配置文件）
  --server-check-interval N 服务器连接检测间隔秒数（覆盖配置文件）
  --no-check-server        禁用服务器连接断开检测
  --log-file PATH          日志文件路径（覆盖配置文件）
  --webhook-url URL        Webhook 通知地址（覆盖配置文件）
  --restart-command CMD    掉线后自动重启 Minecraft 的完整命令行

【配置文件 config.json】

  配置文件集中管理所有参数，支持命令行覆盖。关键字段：

  "heatbeat_interval": 3            → 心跳间隔（秒）
  "heartbeat_timeout": 15           → 心跳超时判定（秒）
  "reconnect_interval_min": 2       → 重连起始间隔（秒）
  "reconnect_interval_max": 30      → 重连最大间隔（秒）
  "startup_grace_period": 90        → 启动宽限期（秒）
  "server_check_interval": 10       → 服务器连接检测间隔（秒）
  "server_check_consecutive": 2     → 连续检测 N 次断开才触发
  "webhook_url": ""                 → Webhook 通知地址（可选）
  "log_file": "afk_monitor.log"     → 日志文件（可选）
  "instance_a": {                   → 实例 A 预设
    "port": 18888,
    "peer_port": 18889,
    "auto_index": 0
  },
  "instance_b": {                   → 实例 B 预设
    "port": 18889,
    "peer_port": 18888,
    "auto_index": 1
  }

  注意：如需本地覆盖而不提交到 Git，创建 config.local.json，
  使用 --config config.local.json 启动即可（已在 .gitignore 中排除）。

【工作原理】

  1. TCP 心跳机制
     两个脚本实例在 127.0.0.1 上通过指定端口建立 TCP 连接
     每秒交换 HEARTBEAT 消息，超时时判定对方掉线

  2. 告别消息协议
     PEER_DOWN → 本地 MC 进程退出，通知对方立即终止
     SHUTDOWN  → 脚本正常退出（Ctrl+C），通知对方正常关闭
     ALIVE     → 常规心跳包

  3. 服务器连接检测
     定期检测 MC 进程的 TCP 远程连接状态
     排除本地回环和认证/CDN 等非游戏端口
     连续 N 次未检测到游戏服务器连接 → 判定掉线

  4. 掉线处理
     一方掉线 → 通知对方 → 对方结束本地 MC 进程
     ← 对方收到通知 → 结束本地 MC 进程

  5. 安全退出
     Ctrl+C → 发送 SHUTDOWN → 通知对方 → 优雅退出
     对方收到 SHUTDOWN → 结束本地 MC 进程

【使用场景与流程示例】

  场景 1：普通挂机
    ① 打开两个 HMCL/PCL/BakaXL 启动器，启动两个 MC 客户端
    ② 运行 start_instance_a.bat → 窗口 1 监控 PID X
    ③ 运行 start_instance_b.bat → 窗口 2 监控 PID Y
    ④ 两个 CMD 窗口保持开启即可挂机
    ⑤ 一方崩溃 → 另一方自动关闭 → 两个 MC 都退出

  场景 2：排位/副本互保
    ① 同上启动两个 MC 客户端
    ② 分别启动脚本
    ③ 如一人掉线，另一人 MC 也会退出 → 避免单人在线

  场景 3：服务器维护/重启
    ① 正常挂机中
    ② 服务器踢出所有玩家
    ③ 脚本检测到服务器连接断开 → 结束本地 MC 进程 → 通知对方

【常见问题】

  Q: 脚本提示"未检测到 Minecraft 进程"
  A: 确保 MC 客户端已完全启动（进入游戏世界），使用 --list 查看

  Q: pid 绑定错误（实例 A 绑了实例 B 的 MC）
  A: 使用 --list 确认 PID 排序，必要时用 --pid 手动指定

  Q: psutil.AccessDenied 权限不足
  A: 右键 CMD → 以管理员身份运行，脚本会自动回退到 netstat 方案

  Q: 防火墙阻止本地 TCP 连接
  A: 本地回环 127.0.0.1 通常不会被防火墙拦截，如遇到请在防火墙
     中放行 python.exe 或相应端口

  Q: 如果想自定义端口
  A: 编辑 config.json 或使用命令行参数 --port / --peer-port

  Q: 如何开启 Webhook 通知？
  A: 在 config.json 中设置 "webhook_url" 为你的 Webhook 地址
     （支持 Discord、企业微信、飞书等 webhook 格式）

  Q: 如何实现掉线后自动重启 Minecraft？
  A: 使用 --restart-command 参数，例如：
     python afk_monitor.py --instance a --restart-command "start mc.bat"
